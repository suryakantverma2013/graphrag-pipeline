# ----- Local ML runtime (GPU) ------------------------------------------------
# PIN the +cu128 LOCAL versions explicitly. The cu128 index tops out at
# torch 2.11.0+cu128; PyPI carries a HIGHER base (2.12.0, CPU-only), so an
# unpinned `torch` lets a transitive dep (torchvision via easyocr) upgrade to
# 2.12.0+cpu and silently kill CUDA (RISK-D). Exact +cu128 pins can't be
# substituted by a CPU wheel. torchvision 0.26.0 is the match for torch 2.11.0.
torch==2.11.0+cu128                 # CUDA build from the index above (D5/RISK-D)
torchvision==0.26.0+cu128          # matched pair; pulled by easyocr — pin it too
# Cap below 5.0: FlagEmbedding 1.4.0 still calls tokenizer.prepare_for_model,
# which transformers 5.x removed (XLMRobertaTokenizer has no attribute
# prepare_for_model at rerank time). transformers 4.x satisfies docling-ibm-models
# (>=4.42) too. Also note: sentence-transformers SEGFAULTS on import in this env,
# so FlagEmbedding is the reranker loader, not the ST CrossEncoder alternative.
transformers<5.0                  # reranker (FlagEmbedding) + Docling backbone
docling[easyocr]>=2.59            # parsing (DocLayNet/TableFormer) + EasyOCR (D6)
FlagEmbedding                     # BGE cross-encoder reranker (bge-reranker-v2-m3)
